Project Handoff Guide

A concise single-page guide for transferring context, decisions, and next actions from one team or session to another.

Audience: Contributors Format: Static HTML Status: Draft

Overview

This document captures the essential context needed to continue work without re-discovery. It focuses on outcomes, constraints, important decisions, and the next useful checkpoint.

Use this format when continuity matters. Keep the page short enough to scan, but complete enough that the next reader can act.

1

Collect Context

Summarize the goal, current state, known constraints, and any external dependencies.

2

Record Decisions

List the meaningful choices already made, including why each one was selected.

3

Define Next Actions

Write concrete next steps with enough detail for another person to continue immediately.

Inputs

  • Current objective and scope boundaries
  • Links to source material or working files
  • Open questions and decision owners

Outputs

  • Readable status summary
  • Decision log with rationale
  • Prioritized next action list
