# Confluence Sync

Downloads an entire Confluence space (or specific pages with descendants) to local Markdown files with YAML frontmatter. Uses the `mcp atlassian` tool as its data source.

**Source:** `tools/mcp-cli/confluence-sync.js`

## Quick Reference

```bash
# Get your cloudId
mcp atlassian getAccessibleAtlassianResources

# List spaces
mcp atlassian getConfluenceSpaces --cloudId <cloudId>

# Sync a space by key
node confluence-sync.js <cloudId> <spaceKey> [outputDir]

# Sync a space by numeric ID
node confluence-sync.js <cloudId> <spaceId> [outputDir] --spaceId
```

## How It Works

1. Resolves the space (by key or `--spaceId`).
2. Fetches all top-level pages via `getPagesInConfluenceSpace`.
3. For each top-level page, recursively fetches descendants via `getConfluencePageDescendants`.
4. Saves each page as a `.md` file with YAML frontmatter.

## Output Structure

```
confluence-<spaceKey>/
  TopPageTitle/
    TopPageTitle.md            # Page content + YAML frontmatter
    ChildPage/
      ChildPage.md
      Grandchild/
        Grandchild.md
    ChildPage.error.txt        # Created on fetch failure
```

## YAML Frontmatter

Generated by `buildFrontmatter()` — includes `id`, `title`, `type`, `status`, `spaceId`, `parentId`, `createdAt`, `version`.

## Content Cleaning

`cleanContent()` strips Confluence-specific `<custom>` tags (emoji, mention, date, placeholder) and blob image references from the markdown body.

## Utility Functions

| Function          | Purpose                                   |
| ----------------- | ----------------------------------------- |
| `mcp(tool, args)` | Wrapper around `mcp atlassian` CLI calls  |
| `sanitize(title)` | Converts page title to safe filename      |
| `stripAnsi(str)`  | Removes ANSI escape codes from CLI output |
| `yamlEscape(val)` | Escapes values for YAML frontmatter       |

## Error Handling

- If a page fetch fails, an `.error.txt` file is written instead of `.md`.
- Missing spaces exit with code 1 and an error message.
- The sync continues even if individual pages fail.

## When Modifying This Code

- It's a standalone Node.js ESM script (`"type": "module"` in package.json).
- Depends on `mcp atlassian` being available (npm-linked from `tools/mcp-cli/`).
- Uses `child_process.execSync` for MCP calls — no async orchestration.
- Run with: `node tools/mcp-cli/confluence-sync.js <cloudId> <spaceKey> [outputDir]`
